# Peer Review 실행 가이드

> Last Updated: 2026-07-15

코드/diff 리뷰에 사용하는 가이드입니다. 계획 문서 리뷰는
`agent-plan-review.md`를 사용합니다.

## 이 문서를 볼 때

- 커맨드 동작·프롬프트·옵션이 바뀌었을 때
- 매니페스트나 템플릿이 바뀌었을 때
- 커밋이나 PR 생성 전 최종 품질 점검이 필요할 때
- PR 리뷰를 수행할 때

## 기본 리뷰 관점

| 관점 | 초점 |
| --- | --- |
| Critic | 기존 저장소 패턴(얇은 래퍼)에 맞는가, 불필요한 복잡도가 추가되지 않았는가 |
| Correctness | `gh` 호출·jq 필터·타임라인 정렬이 실제로 의도대로 동작하는가 |
| Convention | 이름, 커밋 형식, 문서, 한글 작성 규칙을 따르는가 |
| Security | 토큰·시크릿 노출, `allowed-tools` 과확대, 대상 저장소 변경 호출이 없는가 |
| Docs | 동작 변경에 맞춰 README·AGENTS·`.claude/docs/`가 갱신됐는가 |

## 리뷰 초점 (심각도 순)

1. 정확성 버그: 잘못된 `gh` 호출, 깨진 정렬/페이징, 커서 손상
2. 토큰·시크릿 노출, `allowed-tools` 불필요한 확대
3. 얇은 래퍼 원칙 위반(무단 의존성·실행 파일·설정 시스템 추가)
4. 매니페스트 불일치(`plugin.json` ↔ `marketplace.json`)
5. 문서 갱신 누락(동작이 바뀌었는데 README/AGENTS 미갱신)
6. 프롬프트 모호성: 해설 지시문이 "나열"로 퇴화할 여지

## 리뷰 프롬프트 템플릿

```text
repo-walk의 현재 diff를 critic, correctness, convention, security, docs
관점에서 리뷰하라. 잘못된 gh 호출/정렬, 토큰 노출, allowed-tools 과확대,
얇은 래퍼 위반, 매니페스트 불일치, 문서 갱신 누락, AGENTS.md 및
.claude/docs/*.md 위반에 집중하라. 발견 사항을 파일·라인 참조와 함께
심각도 순으로 반환하라. 없으면 남은 검증 리스크를 명시하라.
```

## 출력 규칙

- 발견 사항을 심각도 순으로 먼저 제시합니다.
- 파일과 라인 참조를 포함합니다.
- 칭찬은 나열하지 않습니다.
- 요약보다 구체적 발견이 우선입니다.
- 실제 저장소로 `/repo-walk`를 실행해 확인하지 않았다면 그 공백을 명시합니다.
