# 에이전트 보안 가이드

> Last Updated: 2026-07-15

토큰, 자격 증명, 사용자 데이터를 다룰 때 사용하는 문서입니다. 이 플러그인은
얇은 래퍼라 보안 표면이 작지만, 작아도 지킵니다.

## 토큰 / 시크릿

- 다음을 절대 커밋하지 않습니다:
  - GitHub 토큰(`gho_...`, `ghp_...`), `.env`, `*.pem`, `*.key`
  - 사용자별 로컬 파일, 인증 캐시
- 시크릿 값을 코드, 커맨드 마크다운, 로그, PR 본문, 커밋 메시지에 넣지
  않습니다.
- 데이터 접근은 **사용자의 `gh` 인증을 재사용**합니다. 플러그인이 자체 토큰을
  요구하거나 저장하지 않습니다.

## 커맨드 프롬프트

- 커맨드 마크다운의 `allowed-tools`는 필요한 최소 범위만 부여합니다
  (현재 `Bash(gh:*), Read, Write`). 임의로 넓히지 않습니다.
- 커맨드가 실행하는 `gh` 호출은 읽기 위주여야 합니다. 대상 저장소를
  변경하는 `gh` 호출(issue/PR 생성, push 등)을 순회 로직에 넣지 않습니다.

## 사용자 데이터

- 커맨드가 만드는 상태 파일(`.repo-walk/*.json`)에는 토큰·개인정보를 기록하지
  않습니다. 대상 저장소가 private일 수 있으므로 이 파일은 `.gitignore`로 반드시
  제외하고 커밋하지 않습니다.
- 대상 저장소가 private일 수 있으므로, 가져온 본문·diff를 외부로 전송하지
  않습니다. 처리는 로컬과 사용자의 Claude 세션 안에서만 이루어집니다.

## 커밋 전 확인

- `git diff --check`와 함께 diff에 토큰·시크릿·개인정보가 없는지 눈으로
  확인합니다.
