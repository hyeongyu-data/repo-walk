# Plan Review 실행 가이드

> Last Updated: 2026-07-15

구현 전 계획(plan)을 리뷰할 때 사용하는 가이드입니다. 코드나 diff 리뷰는
`agent-peer-review.md`를 사용합니다.

## 이 문서를 볼 때

- 새 커맨드 옵션·순회 모드·해설 방식 추가
- 여러 파일에 걸친 변경 또는 매니페스트 구조 변경
- 프롬프트를 크게 바꾸는 실험
- "계획 리뷰"나 모호성 점검이 명시적으로 요청된 경우

## 리뷰 질문

1. 목표가 명확하고 범위가 한정되어 있는가?
2. 비목표(non-goal)가 범위 확장을 막을 만큼 명시적인가?
3. 영향받는 컴포넌트(커맨드/매니페스트/문서)가 정확한가?
4. 구조 변경과 동작 변경이 분리되어 있는가?
5. 검증 방법이 구체적인가? (JSON 유효성, frontmatter 확인, 실제 저장소로
   `/repo-walk` 실행 등 실행 가능한 절차인가)
6. 얇은 래퍼 원칙을 지키는가? 새 의존성·실행 파일 없이 `gh`와 프롬프트로
   해결되는가? (YAGNI)
7. 토큰·시크릿 취급과 `allowed-tools` 범위가 안전한가?
8. 문서 갱신(README/AGENTS/`.claude/docs/`)이 계획에 포함되어 있는가?
9. 요청된 결과에 비해 과잉 설계된 단계는 없는가?

## 출력 형식

1. Blocking 이슈
2. Non-blocking 개선 사항
3. 모호한 부분과 가정
4. 제안하는 계획 수정

계획이 타당하면 그렇다고 말하고, 남아 있는 가장 큰 리스크를 나열합니다.
