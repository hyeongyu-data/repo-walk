# 에이전트 금지 사항

> Last Updated: 2026-07-15

이 저장소에서 에이전트가 해서는 안 되는 일의 목록입니다.

## Git / GitHub

- `main`에 직접 push하지 않습니다. 모든 변경은 PR을 거칩니다.
- 사용자 확인 없이 GitHub 원격에 영향을 주는 작업을 하지 않습니다:
  issue 생성·수정·close, label 변경, PR 생성·수정·**merge**, 원격 branch
  push·삭제, 저장소 설정 변경.
- 사용자 승인 없이 PR을 머지하지 않습니다.
- 컨벤션 외 커밋 type을 사용하지 않습니다. 허용:
  `feat`, `fix`, `exp`, `docs`, `refactor`, `test`, `chore`.
- 이미 push된 브랜치를 사용자 동의 없이 force push하지 않습니다
  (rebase 후 `--force-with-lease`는 예외).

## 코드 / 설계

- **얇은 래퍼 원칙을 깨지 않습니다.** 새 의존성·실행 파일·설정 시스템을
  사용자 승인 없이 추가하지 않습니다. `gh` 하나로 되는 일을 새 도구로
  대체하지 않습니다 (YAGNI).
- 요청된 변경에 필요하지 않은 광범위한 리팩터링을 하지 않습니다.
- 구조 변경(파일 이동·이름 변경)과 동작 변경을 한 커밋에 섞지 않습니다.
- 토큰, API 키, 시크릿을 커맨드 마크다운이나 문서에 하드코딩하지 않습니다.
  데이터 접근은 사용자의 `gh` 인증을 재사용합니다.
- `plugin.json`과 `marketplace.json`의 이름·설명을 서로 어긋나게 두지
  않습니다.

## 문서

- 동작·명령어·옵션이 바뀌었는데 관련 `.md`(README, AGENTS, `.claude/docs/`)를
  갱신하지 않은 채 작업을 끝내지 않습니다.
- 규칙 문서 간 충돌하는 내용을 새로 만들지 않습니다. 충돌을 발견하면
  사용자에게 보고합니다.
- 모든 `.md` 파일과 주석은 한글로 작성합니다 (배포물 포함). 임의로 영어로
  되돌리지 않습니다.
